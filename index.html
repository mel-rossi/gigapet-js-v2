DevTools Demo Panel

Message Logging

View Messages Logged by the Browser

Filter Messages

Click "Populate Filters" to write one of each message type to the console, then use the DevTools filter bar to practice each filter technique.

  • Filter by Log Level: Use Default levels dropdown in Console toolbar.
  • Filter by Text: Type GigaPet in the filter bar to only show messages containing that text.
  • Filter by Regular Expression: Type /weight|happiness/i in the filter bar to only show messages containing those terms.
  • Filter by Message Source: Select a File Name in the Messages dropdown in the sidebar to filter by its source.
  • Filter by User Messages: Select User Messages from sidebar to filter by User Messages.

Reproduce a Bug & Apply a Fix

Click "Reproduce Bug" to trigger a bug in the console. Watch from the console and click "Apply Fix" to fix the bug.

Sources, Breakpoints & Check Variables

Open the Sources panel.

The Sources panel is divided into three main sections: the Page tab with the file tree on the left, the Code Editor on the right (or middle), and the Debugger below the other two (or to the rightmost of them).

Check the Event Listener Breakpoints > Mouse > click to pause execution when a click event is triggered. (Click on any button)

Click on "Pause on Breakpoint" to trigger a line-of-code breakpoint.
